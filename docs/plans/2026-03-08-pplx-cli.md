# pplx-cli 实现计划（CLI 版本）

**日期**：2026-03-08  
**项目**：`github.com/toby1991/pplx-cli`  
**目标**：通过 macOS AX UI 自动化控制 Perplexity Desktop App，暴露为直接可用的 CLI 工具  
**状态**：已完成。后续扩展为 MCP Server + API 双后端架构，详见 [MCP 设计文档](2026-03-09-mcp-api-design.md)。

---

## 设计决策：CLI vs MCP

最初选择 CLI 而非 MCP Server 的原因：

- **MCP** 是让 LLM（如 Claude）调用 Perplexity，中间多一层
- **CLI** 直接使用：`pplx "量子计算是什么"` → 立即得到答案
- CLI 支持 shell pipe，可集成到任意脚本
- 不依赖 Claude Desktop 或其他 MCP 宿主

> **后续演进**（2026-03-09）：增加了 `pplx mcp` 子命令作为 MCP Server，`pplx api` 子命令直接调用 Sonar REST API。CLI 和 MCP 共存于同一二进制，共享 driver 层。

## CLI 使用方式

```bash
# 一次性查询（主用途）
pplx "what is quantum computing"
pplx --model "Claude Sonnet" "explain entanglement"
pplx --sources web,academic "quantum computing papers"

# stdin pipe（自动检测）
echo "what is Go?" | pplx
cat question.txt | pplx

# 输出 pipe（自动去色、纯文本）
pplx "top 5 Go CLI libraries" | grep cobra
pplx "what is X" --json | jq '.answer'
pplx "what is X" --json | jq '.citations[].url'

# 交互式 REPL（无参数进入）
pplx

# 实用子命令
pplx status          # 查看当前模式/模型（读 UserDefaults）
pplx models          # 列出所有可用模型
pplx version         # 版本信息
```

## Task 2: CGo AX 自动化层

### `automation/ax.m`（Objective-C）

C 函数接口：

```c
int  ax_activate_app(const char *bundle_id);
void* ax_find_button(const char *description);
int  ax_click(void *element_ref);
char* ax_read_clipboard(void);
void ax_free_string(char *s);
void* ax_wait_for_button(const char *description, int timeout_ms, int poll_ms);
int  ax_open_file_dialog_and_input(const char *path);
char* ax_read_defaults(const char *app_id, const char *key);
void ax_release_element(void *element_ref);
```

**关键实现**：
- 用 `NSWorkspace` 通过 Bundle ID 找 PID
- `AXUIElementCreateApplication(pid)` 创建 App AX 引用
- 递归遍历 `kAXChildrenAttribute` 查找按钮
- 匹配字段：`kAXDescriptionAttribute`
- 点击：`AXUIElementPerformAction(kAXPressAction)`
- 剪贴板：`NSPasteboard.generalPasteboard`
- `ax_wait_for_button`：`usleep(poll_ms * 1000)` 循环轮询

## 已知限制

1. **需要 Accessibility 权限**：在「系统设置 → 隐私与安全性 → 辅助功能」中授权运行终端（iTerm2/Terminal.app）
2. **搜索耗时 10-30 秒**：AX 自动化等待 Perplexity 完成加载，有 spinner 提示
3. **剪贴板临时覆盖**：读取结果时覆盖用户剪贴板，搜索完成后自动恢复原始内容
4. **串行执行**：每次只能有一个搜索进行中（Perplexity App 本身限制）
5. **macOS only**：使用 AXUIElement 和 IOKit API，不支持跨平台
6. **Headless 需虚拟显示器**：UI 后端需要至少一个物理或虚拟显示器，且不能锁屏
